# 04 — Decision Log (ADR)

> Журнал архитектурных и продуктовых решений: проблема → варианты → выбор → причина + источники. Накапливается, не переписывается. Формат: ADR-NNN.

## ADR-001 — Версия Unity: 6000.4.9f1 (изначально рекомендована 6.3 LTS)
**Дата:** 2026-06-10 · **Статус:** принято, дополнено в тот же день

**Проблема:** выбрать версию Unity для проекта со сроком сдачи (диплом).

**Варианты:**
1. Unity 6.0 LTS — поддержка до октября 2026 (кончится до защиты).
2. **Unity 6.3 LTS (6000.3.17f1)** — текущая LTS, поддержка до декабря 2027.
3. Unity 6.4 (Supported) — новые фичи (встроенный ECS, DirectStorage), но поддержка умрёт с выходом 6.5.

**Рекомендация ресёрча:** Unity 6.3 LTS — официальная рекомендация Unity для новых проектов; двухлетний хвост багфиксов покрывает весь срок диплома.

> **UPD 2026-06-10 — итоговое решение:** пользователь уже создал проект на **Unity 6000.4.9f1 (6.4, Supported)**. Остаёмся на 6.4: проект инициализирован, пакеты совместимы (URP 17.4, Cinemachine 3.1.6, Input System 1.19, AI Navigation 2.0.12), EditMode-тесты 3/3 зелёные. **Принятый риск:** поддержка 6.4 закончится с выходом 6.5 (~конец 2026); при блокирующих багах возможен переход на 6.3 LTS. Версию редактора не меняем до конца диплома.

Источники: [support schedule](https://unity.com/releases/unity-6/support) · [6.3 LTS blog](https://unity.com/blog/unity-6-3-lts-is-now-available)

## ADR-002 — Cinemachine 3.1, не 2.x
**Дата:** 2026-06-10 · **Статус:** принято (Фаза 0)

**Проблема:** Cinemachine 3.x кардинально сменил API относительно 2.x (большинство туториалов в сети — про 2.x).

**Выбор:** Cinemachine **3.1.x** — verified-версия для Unity 6.3. Работать только по документации 3.x: namespace `Unity.Cinemachine`, единый `CinemachineCamera` вместо `CinemachineVirtualCamera`/`FreeLook`, pipeline-компоненты как обычные MonoBehaviour, Channels вместо слоёв. **Причина:** 2.x — legacy; начинать новый проект на устаревшем API бессмысленно.

Источник: [Upgrade guide 2.x→3.x](https://docs.unity3d.com/Packages/com.unity.cinemachine@3.1/manual/CinemachineUpgradeFrom2.html)

## ADR-003 — UI: uGUI (Canvas + TMP) для HUD, UI Toolkit не используем в рантайме
**Дата:** 2026-06-10 · **Статус:** принято (Фаза 0)

**Проблема:** uGUI vs UI Toolkit для игрового UI (RTS HUD: миникарта, панель выделения, health bars над юнитами; TPS HUD: прицел, кулдауны; меню/настройки).

**Варианты:**
1. **uGUI всё** — зрелый, огромная база рецептов (миникарта = RawImage+RenderTexture, health bars = world-space canvas), твинится PrimeTween, дружит со сторонними ассетами.
2. UI Toolkit всё — современный стек, но world-space UI появился только в 6.2/6.3 (свежая фича = риск).
3. Гибрид (uGUI HUD + UI Toolkit меню) — две системы = двойная сложность стилей/анимаций.

**Выбор:** uGUI для всего. **Причина:** минимизация рисков в дедлайн-проекте; TextMeshPro теперь встроен в uGUI 2.0; официальная позиция Unity допускает uGUI как полностью поддерживаемый путь. Пересмотрим, только если упрёмся в производительность data-heavy экранов (не ожидается).

Источники: [UI comparison](https://docs.unity3d.com/Manual/UI-system-compare.html) · [World-space UI Toolkit](https://docs.unity3d.com/6000.3/Documentation/Manual/ui-systems/world-space-ui.html)

## ADR-004 — Твининг: PrimeTween вместо DOTween
**Дата:** 2026-06-10 · **Статус:** принято (Фаза 0)

**Варианты:** DOTween (популярнейший, но closed-source, аллокации, NRE при уничтожении целей) / **PrimeTween** (MIT, zero-allocation, безопасен при destroy) / LitMotion (MIT, быстрейший, data-oriented).

**Выбор:** PrimeTween. **Причина:** open-source MIT (чистая лицензионная история для диплома), zero-alloc (плюс к разделу о производительности), API проще LitMotion для UI-juice. DOTween отвергнут из-за closed-source и известных NRE-ловушек.

Источники: [PrimeTween](https://github.com/KyryloKuzyk/PrimeTween) · [Tween benchmark](https://github.com/AnnulusGames/TweenPerformance)

## ADR-005 — ИИ юнитов: собственная FSM/utility-AI, пакет Behavior не используем
**Дата:** 2026-06-10 · **Статус:** принято (Фаза 0)

**Проблема:** com.unity.behavior (behavior trees) с мая 2026 переведён в maintenance-only (команда сокращена ещё в феврале 2025).

## ADR-006 — Аудио: только CC0/CC-BY библиотеки + локальная генерация; все облачные сервисы отвергнуты
**Дата:** 2026-06-10 (пересмотрено в тот же день под ADR-009) · **Статус:** принято (Фаза 0)

**Проблема:** выбрать пайплайн музыки, SFX и голосов юнитов с чистыми лицензиями при **нулевом бюджете** (ADR-009: без подписок, триалов, карт).

**Варианты и почему отпали:**
- **Udio** — скачивание треков отключено на всех тарифах после сделки с UMG → непригоден физически.
- **Suno** — нет официального API; free-тир без скачивания (сделка с Warner); платный Pro запрещён ADR-009.
- **ElevenLabs** (TTS/SFX/музыка) — freemium-подписка → запрещён ADR-009. *(Первая редакция этого ADR рекомендовала Starter $6/мес — отменено.)*
- **Stable Audio API / Azure TTS F0** — платный API / требует облачный аккаунт → запрещены ADR-009.

**Выбор (всё $0, без аккаунтов):**
- Музыка: incompetech/Kevin MacLeod (CC-BY 4.0, кредит в титрах) + Pixabay + OpenGameArt (CC0/CC-BY); опционально локальная генерация **Stable Audio Open** (открытые веса, Stability Community License — свободно при < $1M выручки).
- SFX: Kenney (CC0) + Sonniss GDC 2026 (royalty-free) ≈ 90%; точечно Freesound (бесплатный API, фильтр CC0/CC-BY); ретро/UI — sfxr/Bfxr/ChipTone локально.
- Голоса юнитов: путь №1 — **Kenney Voiceover Pack / (Fighter)** (CC0, готовые ack-реплики); путь №2 — **Piper TTS** локально (код GPL, лицензию конкретного голоса проверять на HF rhasspy/piper-voices).
- **MusicGen** — только запасной вариант: веса **CC-BY-NC** (для учебного диплома допустимо, для коммерческого распространения — нет; фиксирую согласно ADR-009/zero-budget §Лицензии).

**Причина:** соответствие нулевому бюджету; лицензионная чистота для публичного репозитория; Kenney Voiceover закрывает голоса вообще без генерации.

Источники: см. [[Research/00 - Prerequisites|Prerequisites, раздел B]]

## ADR-007 — Арт: CC0-набор Kenney + Quaternius; платные паки исключены
**Дата:** 2026-06-10 (пересмотрено в тот же день под ADR-009) · **Статус:** принято (Фаза 0)

**Выбор:** Kenney (Blaster Kit, Space Kit, UI Pack Sci-Fi, Particle Pack) + Quaternius (Animated Mech, Tanks, Ultimate Modular Sci-Fi) — всё CC0, можно коммитить в публичный репозиторий. Добор: Poly Pizza, Sketchfab (фильтр CC0); прототип-геометрия — примитивы + ProBuilder (бесплатный пакет Unity). Шрифты Russo One + Exo 2 (OFL, кириллица есть; Orbitron отвергнут — нет кириллицы). Иконки game-icons.net (CC-BY 3.0, атрибуция) + Kenney UI (CC0).

**Synty POLYGON ($49.99) — отвергнут** по ADR-009 (нулевой бюджет). Дополнительный аргумент: EULA Synty всё равно запрещает исходники в публичном репозитории. Бесплатные ассеты Unity Asset Store — тоже под Asset Store EULA (исходники в публичный репо нельзя), поэтому предпочтение строго CC0-источникам.

**Причина:** бюджет $0; юридически чистый открытый репозиторий; Kenney и Quaternius стилистически совместимы (low-poly).

Источники: см. [[Research/00 - Prerequisites|Prerequisites, раздел C]]

## ADR-008 — VCS: выборочный LFS с первого коммита; CI — минимальный и опциональный
**Дата:** 2026-06-10 · **Статус:** принято (Фаза 0), развилка по CI за пользователем

**Выбор:**
- LFS включаем до первого коммита (миграция задним числом = переписывание истории), но **выборочно**: крупные бинарники в LFS, мелкие (<~5 МБ png/ogg) — обычный git. Причина: free-тир 10 GiB хранилища + 10 GiB трафика/мес; data packs отменены; каждый clone/CI-checkout жжёт трафик.
- Билд распространяем через **GitHub Releases** (2 GiB/файл, трафик бесплатен), не через LFS.
- CI (GameCI unity-builder@v4): Actions для публичных репо бесплатны, но нужен секрет UNITY_LICENSE (действие пользователя) и сборки медленные. ~~Рекомендация: стартуем без CI~~ → **UPD 2026-06-10: пользователь решил делать CI/CD.** Внедрено: `tests.yml` (EditMode+PlayMode на push/PR) + `build-release.yml` (Windows x64 по тегу `v*` → автоматический GitHub Release). Основной путь сборки всё равно локальный (Project Forge); CI — верификация и релизы. До добавления секретов `UNITY_LICENSE`/`UNITY_EMAIL`/`UNITY_PASSWORD` workflow будет падать — ожидаемо.

Источники: см. [[Research/00 - Prerequisites|Prerequisites, раздел E]]

## ADR-009 — Жёсткое ограничение: нулевой бюджет проекта
**Дата:** 2026-06-10 · **Статус:** принято (директива пользователя, `zero-budget-constraint.md`) · **Приоритет: выше всех остальных решений**

**Суть:** бюджет проекта — $0. Запрещены: покупки, подписки, платные API, платные ассеты, а также **любые триалы/freemium с картой, сроком или конвертацией в платное**. Разрешены только: FOSS (бесплатно навсегда, без карты), локальная/self-hosted генерация, CC0/свободные ассеты, постоянно-бесплатные сервисы без карты (Unity Personal, GitHub, Git LFS free tier, gh CLI, Obsidian, Node.js).

**Последствия для ранее принятых решений:**
- ADR-006 пересмотрен: ElevenLabs/Stable Audio API/Azure исключены → CC0/CC-BY библиотеки + локальная генерация (Stable Audio Open, Piper).
- ADR-007 пересмотрен: Synty исключён → строго CC0-арт.
- Лицензионная политика: приоритет CC0 > CC-BY (с атрибуцией) > CC-NC (только если нет свободного аналога, помечать отдельно — для учебной работы допустимо, для коммерческого распространения нет). Всё внешнее — в `Licenses & Attribution.md`; ничего с неясной лицензией.

**Проверка перед выбором любого инструмента:** бесплатен навсегда? без карты? не триал? Если бесплатен только trial — отклонить и найти FOSS/CC0-аналог с фиксацией здесь.
