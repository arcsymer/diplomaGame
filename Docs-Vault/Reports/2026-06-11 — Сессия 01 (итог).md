## Самые ценные пойманные баги

### 1. Cinemachine 3.x — `CinemachineRotationComposer` (M1)
**Баг:** `CinemachineRotationComposer` не имеет свойства `LookAt`. Паттерн из документации 2.x (установить цель на компоненте) не компилируется.
**Фикс:** цель задаётся на самой `CinemachineCamera`, не на pipeline-компоненте.
**Ценность:** без этого знания TPS-камера не завелась бы на первом M1.

### 2. Cinemachine 3.x — `Priority` как структура (M1)
**Баг:** `CinemachineCamera.Priority` — это `PrioritySettings`, не `int`. Присваивание `int` работает (неявная конверсия), но сравнение в `Assert.AreEqual` давало false.
**Фикс:** сравнивать через `.Priority.Value`.
**Ценность:** PlayMode-тест поймал до того, как логика переключения камер вошла в release.

### 3. TMP — абстрактный класс `TMP_Text` (M6)
**Баг:** `AddComponent<TMP_Text>()` молча возвращает null — класс абстрактный.
**Фикс:** использовать `TextMeshProUGUI`.
**Ценность:** Forge-кнопки Setup HUD строили HUD без текстовых компонентов; баг бы всплыл в рантайме необнаруженным.

### 4. TMP Essential Resources — async import в batch-режиме (M6)
**Баг:** `AssetDatabase.ImportPackage` в batch-режиме (`-executeMethod`) асинхронен и не успевает до завершения скрипта. HUD строился без TMP-ресурсов → NRE в `SetText` в рантайме.
**Фикс:** распаковать `.unitypackage` напрямую (это tar.gz) с сохранением `.meta`/GUID, минуя Import API.
**Ценность:** устранил непредсказуемую зависимость от порядка выполнения Editor-кода.

### 5. Юниты не атаковали здания (M10 — критический)
**Баг:** `UnitCombat.ScanForTarget()` искал цели только через `UnitRegistry`. Здания в нём не регистрировались. Волна противника доходила до HQ игрока, стояла рядом и ничего не делала. Победа через атаку была невозможна.
**Фикс:** рефакторинг на цели-`Health` из обоих реестров (`UnitRegistry` + `BuildingRegistry`); буфер +2м к `attackRange` для зданий (ADR-010).
**Ценность:** без этого фикса игра не имела конечного состояния «Победа» через действия ИИ.

### 6. Толпа на rally-point — ИИ не набирал волну (M10)
**Баг:** 11 из 12 юнитов ИИ вечно находились в состоянии `Moving` (NavMeshAgent формально двигался к rally-point, но реально стоял в толпе). `EnemyCommander` не видел достаточно `idleCount` для запуска волны.
**Фикс:** формационные смещения при спавне + анти-застревание в `Unit.Update` (стоит ≥ 1.5с при `velocity.sqrMagnitude < 0.01` → считается прибывшим).
**Ценность:** без этого фикса ИИ никогда не атаковал после первой волны.

### 7. Float-точность на границе кулдауна (M3)
**Баг:** `now - lastFireTime >= cooldown` давало false при точном равенстве из-за погрешности float.
**Фикс:** введён эпсилон 1e-4 в `FireRateLogic.CanFire`.
**Ценность:** EditMode-тест поймал до рантайма; без фикса стрельба «заедала» ровно на границе кулдауна.

## Методология — что сработало

### Project Forge + batch-CLI
Единственный Editor-тул с идемпотентными кнопками сетапа (Setup M1, Setup M2, ... Setup M9). Каждый `ForgeBatch.SetupMX()` запускается через `-executeMethod` в одну строку CLI. Это позволило:
- воспроизводить любой этап сцены за секунды,
- не держать состояние «в голове» между вызовами,
- тестировать сетап изолированно (PlayMode-тесты не зависят от ручного сетапа).

### Субагентная архитектура разработки
Специализированные субагенты (design → backend → frontend → qa) работали параллельно по независимым слоям. Оркестратор удерживал только архитектурный контекст. Это позволило пройти M0–M11 за один день без потери когерентности между слоями.

### Тесты как сетка безопасности
256 тестов (EditMode + PlayMode) поймали 7 критических и нетривиальных дефектов до того, как они попали в рантайм. Ключевой паттерн: Logic-классы (чистая статика) → EditMode-тест; MonoBehaviour с зависимостями → PlayMode-тест через `InitForTest`. Интеграционные тесты (`MatchSimulationTests`, timeScale×10) нашли M10-BUG-001 автоматически.

### Нулевой бюджет — не ограничение
ADR-009 (нулевой бюджет) сформировал чёткие правила выбора инструментов и ассетов. Kenney CC0 + Kevin MacLeod CC-BY покрыли 100% потребностей в арте и аудио. Ни одного платного ресурса в проекте.
