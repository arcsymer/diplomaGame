### Промпт для Claude Code (модель Fable 5) — v2 (полный продукт + автономия)

## Твоя миссия

Ты — инженер-геймдизайнер, который ведёт мой дипломный проект от пустого репозитория до **законченной, отполированной, играбельной игры со сборкой**. Это 3D-игра на Unity, где игрок **одновременно** управляет армией в режиме **RTS** (вид сверху) и одним героем в режиме **TPS** (вид от третьего лица, стрельба). ## 1. Концепция игры (Design Pillars)

**Жанр:** гибрид RTS + TPS в одной сессии, без загрузок между режимами.

**Ключевая механика** — мгновенное переключение режима одной кнопкой (`Tab`/`F`):
- **RTS (Commander):** камера сверху, выделение (клик, рамка, контрол-группы), приказы (move, attack-move, hold, patrol), строительство, производство, экономика.
- **TPS (Hero):** камера за спиной героя, WASD + мышь, стрельба, 2–4 способности. Герой остаётся юнитом и в RTS-режиме.

**Сниженный микроменеджмент** (предмет диплома — обосновывай каждое решение в Decision Log):
- авто-добыча ресурсов; умный ИИ юнитов (держат строй, сами выбирают цели, отступают при низком HP); армейские/групповые приказы; упрощённая экономика (1–2 ресурса) с rally-point; TPS-герой снижает нужду в высоком APM.

**Цель:** понизить порог входа и требуемый APM против классического RTS, сохранив тактическую глубину.

**Объём контента:** одна играбельная карта-сценарий (база игрока + база ИИ-врага + ресурсы + задача уничтожить врага). Глубина систем и полировка важнее количества юнитов.

## 2. Что значит «игра полностью сделана» (Definition of Done)

Игра не считается готовой, пока всё это не сделано и не связано:

- **Геймплей:** все системы RTS+TPS играбельны, сбалансированы, без явных багов; сценарий проходится до победы и до поражения.
- **UX/UI:** главное меню, экран настроек (графика, звук, управление), пауза, экраны победы/поражения, оба HUD (RTS: миникарта, панель выделения, командная карта, ресурсы; TPS: прицел, HP, кулдауны), тултипы, читаемый фидбек (маркеры выделения и приказов, health bars, индикация урона), «juice» — анимации, переходы, реакция интерфейса.
- **Аудио:** музыка (меню / эмбиент / бой), SFX (клики, приказы, оружие, постройки, попадания, UI), голоса-подтверждения юнитов, единый **AudioMixer**, ползунки громкости в настройках.
- **Арт/VFX:** связный визуальный стиль, освещение, пост-обработка (URP Volume), эффекты (вспышки выстрелов, попадания, взрывы, эффект постройки/спавна).
- **Сборка:** рабочий билд под целевую платформу (напр. Windows `.exe`), приложен к **GitHub Release** с тегом версии.
- **Производительность:** стабильный FPS на целевой машине; без утечек и грубых просадок.

## 3. Технологический стек и пакеты

- **Unity LTS** (точную версию и платформу уточни в Фазе 0), **C#**, чистая архитектура, без god-объектов.
- Ввод: **Input System** с раздельными Action Maps для RTS и TPS.
- Камеры: **Cinemachine** (отдельные виртуальные камеры для RTS-вида и TPS-вида с плавным переходом).
- Навигация: **NavMesh / AI Navigation**.
- Рендер: **URP** + Volume для пост-обработки.
- UI: **TextMeshPro** + uGUI или UI Toolkit (выбери и обоснуй).
- Данные юнитов/зданий/способностей/звуков: **ScriptableObject**-ассеты, не хардкод.
- Аудио: **AudioMixer** с группами (Master/Music/SFX/UI/Voice).
- Тесты: EditMode/PlayMode на ключевую логику (урон, выделение, переключение режимов) — где разумно.
- Версии пакетов — **актуальные под выбранную Unity**; сверяй сам, не хардкодь устаревшее.

## 5. ГЛАВНЫЙ ИНСТРУМЕНТ — единый Editor-тул «Project Forge»

**Критично.** Ты редактируешь только текстовые файлы и **не можешь** кликать в Unity-редакторе, перетаскивать объекты, настраивать Inspector, создавать сцены руками. Поэтому всё, что требует работы в редакторе/сцене/ассетах, ты оформляешь как код-автоматизацию внутри **одного** Editor-окна, которое я запускаю одной кнопкой.

- Один `EditorWindow`, меню `Tools/Project Forge`. **Растёт вместе с проектом** — новая фича = новая кнопка/секция в этом же туле, а не одноразовые скрипты.
- Вкладки (примерно): `Scenes`, `Prefabs`, `Managers`, `Config (SO)`, `NavMesh/Lighting`, `UI`, `Audio`, `VFX`, `Build`, `Validation`, `Reports`.
- Кнопки делают полную настройку через UnityEditor API: `EditorSceneManager`, `PrefabUtility`, `AssetDatabase`, `GameObject.AddComponent`, `SerializedObject`/`SerializedProperty` (для приватных `[SerializeField]`), запекание NavMesh, расстановка спавнов, импорт/привязка аудиоклипов и AudioMixer, сборка UI-канвасов, привязка VFX, **триггер сборки билда** (`BuildPipeline`), и т.д.
- Операции **идемпотентны** где возможно.
- `Validate` — проверка сцены (пропущенные ссылки, наличие менеджеров, корректность настройки).
- `Generate Report` — экспорт статистики и диаграмм в Obsidian.

**Правило:** хочется написать «зайди в Unity и настрой X руками» — вместо этого добавь кнопку в Forge. То, что физически нельзя автоматизировать (активация лицензии, ручной арт), проговаривай явно.

## 9. Дорожная карта (майлстоуны)

Gantt держи в `03 - Roadmap.md`. Порядок:

- **M0 — Инфраструктура:** репо + LFS + `.gitignore`/`.gitattributes`, `CLAUDE.md`, Obsidian-хранилище, Unity-проект из URP-шаблона, установка пакетов через manifest, скелет Project Forge, базовая сцена-песочница, первый коммит.
- **M1 — Режимы и камера:** переключение RTS↔TPS, две Cinemachine-камеры, стейт-машина режима, два Action Map. *(stateDiagram.)*
- **M2 — RTS-управление:** движение по NavMesh, выделение (клик/рамка/контрол-группы), приказы move/attack-move/hold/patrol, rally-points.
- **M3 — TPS-герой:** прямое управление, прицел/стрельба, 2–4 способности, переключение «контроль героя ↔ командование».
- **M4 — Бой и ИИ:** HP/урон/смерть, авто-выбор целей, отступление при низком HP, строй; `Unit` + ScriptableObject-статы.
- **M5 — Экономика и постройки:** 1–2 ресурса, авто-добыча, здания, производство юнитов, очередь.
- **M6 — Полный UX/UI:** главное меню, настройки (графика/звук/управление), пауза, экраны победы/поражения, оба HUD, тултипы, фидбек, juice/переходы.
- **M7 — Аудио:** музыка (меню/эмбиент/бой), SFX, голоса-подтверждения юнитов, AudioMixer, громкость в настройках; импорт и привязка через Forge.
- **M8 — Визуал:** интеграция арт-паков, VFX (вспышки/попадания/взрывы/постройка), освещение, пост-обработка (URP Volume), визуальная связность.
- **M9 — Сценарий:** играбельная карта, ИИ-противник (простой, но осмысленный), условия победы/поражения, поток матча.
- **M10 — Полировка:** баланс, «анти-микро»-доводка умного ИИ и групповых приказов, производительность/FPS, плейтест, фикс багов.
- **M11 — Сборка и релиз:** билд под платформу через Forge (`BuildPipeline`), GitHub Release с тегом и приложенной игрой, README со скриншотами.
- **M12 — Документация диплома:** финализация GDD, архитектуры, Decision Log, метрик, диаграмм, `Licenses & Attribution`; итоговый отчёт.

Каждый майлстоун: краткий план → реализация → кнопка(и) в Forge → отчёт → коммит/тег.
