## 1. ЦИКЛ УЛУЧШЕНИЯ (как работать на каждом круге)

**Measure → Benchmark (vs SC2) → Identify → Prioritize → Experiment → Implement → Verify → Document → Commit → Repeat.**

1. **Measure** — сними метрики текущего состояния (FPS/frame time, input latency, GC-аллокации, размер билда, число багов, винрейты AI vs AI, читаемость, отзывчивость).
2. **Benchmark** — изучи, как соответствующую сферу решает **StarCraft II** (и близкие игры/решения); зафиксируй разрыв.
3. **Identify** — найди конкретные слабые места.
4. **Prioritize** — оцени по матрице **impact × effort**, веди backlog улучшений с приоритетами.
5. **Experiment** — сравни 2–3 **FOSS**-подхода/инструмента, при необходимости прототипируй, измерь.
6. **Implement** — внедри лучший вариант.
7. **Verify** — тесты + профилирование + плейтест (включая авто AI vs AI); не допускай регрессий.
8. **Document** — запиши в Obsidian: решение, причина, источники, **before/after с цифрами**, сравнение с эталоном.
9. **Commit** — закоммить (и тег на вехе).
10. **Repeat** — следующий приоритет / следующая сфера.

Исследуй интернет **в любом объёме** — полный доступ. Любые ассеты/инструменты — только FOSS/CC0/локальные (нулевой бюджет).

## 2. СФЕРЫ УЛУЧШЕНИЯ

Проходи **каждую** сферу, итеративно, по кругу. В каждой — бенчмарк по SC2 и проверка цели «сниженный микро».

### 2.1 Код и инженерия
- Рефакторинг к чистой архитектуре, SOLID, разделение ответственности; устранение god-объектов; снижение связности (event bus / DI / интерфейсы).
- Data-driven: вынос параметров в ScriptableObject/конфиги — добавление контента только данными.
- Единый code style, `.editorconfig`, Roslyn-анализаторы; самодокументируемые имена; осмысленные комментарии.
- `asmdef`-модули для скорости компиляции и изоляции; устранение циклических зависимостей.
- Обработка ошибок, null-safety, защитное программирование; логирование исключений.
- Покрытие тестами ключевой логики; регрессионные тесты на каждый найденный баг.
- Журнал техдолга — систематически гаси.

### 2.2 Архитектура и системы
- Пересмотр стейт-машин/поведенческих деревьев; событийная развязка UI и логики.
- Граф зависимостей без циклов; bootstrapping, scene-management, аддитивные сцены.
- Расширяемость: новые юниты/способности/здания — без правки кода ядра.
- Сохранение/загрузка состояния матча (если в скоупе), надёжная сериализация.
- Архитектурно не закрывай путь к расширению (мультиплеер — вне скоупа диплома, но отметь точки расширения).

### 2.3 Геймплей и механики (vs SC2)
- Качество RTS-петли: отзывчивость выделения, очередь приказов, smart-cast, контрол-группы, rally/waypoints, формации, attack-move и stutter-step feel — эталон SC2.
- Качество TPS: прицеливание, hit-registration, ощущение оружия, движение, камера.
- Переключение RTS↔TPS: бесшовность, сохранение контекста, без джанка.
- Анти-микро: проверь, что фичи **реально снижают APM**, сохраняя тактическую глубину (авто-таргет, отступление, удержание строя, групповые/армейские приказы).
- Макро: продакшн-вкладка, обнаружение idle-воркеров, хоткеи, пейсинг supply/экономики.

### 2.4 Управление, ввод, отзывчивость
- Низкая задержка ввода, input buffering, мгновенный отклик команд (как в SC2).
- Камера: edge/WASD-пан, зум, поворот, камера-букмарки/хоткеи, плавность.
- Полная переназначаемость клавиш, пресеты; геймпад при уместности; консистентные хоткеи командной карты.

### 2.5 Pathfinding, стиринг, коллизии
- Локальное избегание (RVO/ORCA-подобное), флокинг — **без слипания и застреваний** (pathing-эталон SC2).
- Коллизии и расталкивание юнитов, формационное и групповое движение.
- Производительность NavMesh при большом числе агентов; динамические препятствия (здания).

### 2.6 ИИ (юнитов и противника)
- Микро-ИИ юнитов: приоритет целей, фокус-огонь, кайтинг, отступление при низком HP, авто-использование способностей.
- Сквад/армейский ИИ: формации, решение о входе в бой и об отступлении.
- ИИ-командир: билд-ордеры, разведка, тайминги атак, экспанд, реакция на игрока, **масштабирование сложности** (несколько уровней).
- Сравни FSM / behavior tree / utility AI / GOAP — выбери и обоснуй; противник честный, но интересный.

### 2.7 Баланс
- Стат-листы: HP/DPS/стоимость/время постройки/контры (камень-ножницы-бумага); баланс экономики и таймингов экспанда.
- Баланс TPS-героя относительно армии; баланс способностей и кулдаунов.
- Метрики из **авто-плейтестов (AI vs AI)** и ручных тестов; баланс-таблицы в Obsidian/Forge; итеративная подстройка.
- Кривая сложности противника.

### 2.8 Визуал и арт-дирекшн
- Единый стиль, палитра, **читаемость** (цвета фракций/команд — читаемость как в SC2 при хаосе боя).
- Узнаваемые силуэты юнитов, цветокодирование состояний; читаемость при большом числе юнитов.
- Композиция кадра, окружение, доодады, сет-дрессинг, террейн-текстуры, декали.

### 2.9 Освещение, шейдеры, пост-обработка
- Реалтайм + запечённый свет, GI, тени, настроение/время суток.
- Шейдеры: PBR; маска team-color; outline выделения; fog of war; dissolve/warp-in построек.
- URP Volume: bloom, color grading, ambient occlusion, vignette, depth of field — со вкусом, не переборщить.

### 2.10 VFX
- Вспышки выстрелов, снаряды/трейлы, попадания по разным материалам, взрывы, смерти, постройка/спавн, эффекты способностей, окружение.
- VFX Graph (GPU) vs Shuriken — выбирай по нагрузке; **пулинг** эффектов.

### 2.11 Анимация
- Юниты: idle/move/attack/death/ability/build/gather; бленд-деревья, плавные переходы, **вариативность** (без роботизированной униформности армий).
- TPS-герой: локомоция, aim, shoot, reload, способности, hit-reaction.
- Процедурка: IK (стопы, look-at, прицел оружия), рэгдоллы; анимэвенты для синка SFX/VFX.
- 12 принципов: вес, антиципация, follow-through, secondary motion; рекойл/турели/отдача.

### 2.12 Game feel / juice
- Hit-feedback: hit-stop/freeze-frames, screen shake, нокбэк, флэш, партиклы, damage numbers.
- Камера-динамика: shake/kick, look-ahead, плавный фоллоу.
- UI-juice: твины/изинги (DOTween), отклик кнопок, переходы.
- Каждое действие — **мгновенный** аудио-визуальный отклик; отдельный цикл тюнинга констант feel.
- Сатисфакция разрушения (дебри, взрывы); отзывчивость кликов/команд + голоса юнитов (как SC2).

### 2.13 Аудио и музыка
- **Адаптивная музыка**: слои меню/исследование/бой по интенсивности, стингеры и переходы.
- SFX на каждое действие; вариативность оружия, материалы попаданий, UI, эмбиент-беды.
- **Голоса-подтверждения юнитов** (как SC2) — локальный TTS.
- Микс: группы AudioMixer, **дакинг** (музыка под важные SFX), приоритеты, 3D-спатиал, rolloff, реверб-зоны; без клиппинга; настройки громкости.

### 2.14 UI / UX / доступность
- Инфо-архитектура HUD: миникарта, командная карта, ресурсы, supply, очередь, алерты (under attack / unit ready).
- Меню: главное, настройки (графика/звук/управление/геймплей), пауза, победа/поражение, загрузка.
- Онбординг: туториал, тултипы, контекстные подсказки, хинты управления.
- Доступность: colorblind-режимы, переназначение клавиш, субтитры, размер текста, UI-скейл, reduced motion.
- Адаптивность под разрешения/соотношения; hover/disabled/подтверждения; единая дизайн-система; QoL — индикаторы дальности, превью-сетка построек, визуализация waypoints/rally.

### 2.15 Уровень / сценарий
- Лэйаут: чокпоинты, высоты, экспанды, размещение ресурсов, баланс/симметрия.
- Пейсинг сценария, цели, ясность победы/поражения; спавны, fog of war, видимость, разведка.
- Кривая сложности энкаунтеров; средовое повествование; (стретч) дополнительные сценарии/карты.

### 2.16 Производительность и оптимизация
- Профилирование (CPU/GPU/память/GC); **пулинг** (снаряды/юниты/VFX/аудио-источники).
- Бэтчинг (SRP Batcher / GPU instancing / static-dynamic), LOD-группы, окклюжн/фрустум-куллинг.
- Job System / Burst на горячих путях (большое число юнитов); бюджет кадра, **стабильный frame time**.
