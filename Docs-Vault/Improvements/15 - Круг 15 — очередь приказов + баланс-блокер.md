## Tests CI: настоящая причина красноты ≠ баланс-тест (ПОПРАВКА прежнего диагноза)
Прежняя запись винила `BalanceDiagnosticsTests`. По CI-XML это **неверно**: `DiagnosticSeries_12Rounds_LogCorrelations` на CI **проходил** — его единственный assert (`playerWins+enemyWins+draws==12`) тривиально истинен, ничьи его не валят.

**Настоящая причина (Tests CI красный с круга 11, 12.06):** перф-гейт `SceneIntegrationTests.SandboxScene_PerfDuringWave` — `Assert.Less(avgMs, 33f)` (порог 30 FPS) недостижим на headless software-GL раннере GitHub (llvmpipe, GPU нет): `Expected: less than 33.0f / But was: 48.78f` (~20 FPS). Тест добавлен в круге 11 — ровно когда покраснел main. Класс ADR-030: порог под локальное железо нерепрезентативен на CI.

## Три тест-фикса (только тестовые файлы) → main зелёный
- **(A) `ce62797` — SceneIntegration:** перф-порог применяется лишь в интерактивном редакторе; в `Application.isBatchMode` (CI/headless) метрика пишется в `scene-integration.json` и лог, но прогон не гейтится. Локальный регресс-гейт сохранён.
- **(B) `f60e0ae` — BalanceDiagnostics (реальный, но НЕ блокировавший CI баг):** stall-breaker расходился с рабочим в `BalanceSimulationTests`. Порт: цель AttackMove = **база врага** (а не центр масс живых — он давал «зеркальную встречу» армий и цикл ретрит-атака), `SuppressRetreatForStallBreaker()` **после** `IssueCommand` (команда сбрасывает флаг), окно стейлмейта 30→15 с. Итог: **9 ничьих → 0, 12/12 раундов решаются** за 23–65 с.
- **(C) `9beeab2` — OrderQueueLogicTests:** 7 из 21 падали на NavMesh-`LogError` (`SetDestination`/`ResetPath` без NavMesh). `LogAssert.ignoreFailingMessages=true` из `[SetUp]` не действует — UTF 1.6 создаёт LogScope теста **после** SetUp. Фикс: ставим флаг в теле теста (через `CreateTestUnit` — первая строка каждого Unit-теста). Логика очереди по-прежнему ловится `Assert`'ами.
