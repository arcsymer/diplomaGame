## БЛОКЕР: PlayMode баланс-тест красный на main (предшествует кругу 15)
- `DiplomaGame.Tests.Runtime.BalanceDiagnosticsTests.DiagnosticSeries_12Rounds_LogCorrelations` падает на **main** (CI Tests = failure и 13.06, и 22.06). EditMode зелёный; падает PlayMode.
- **Симптом (из CI-лога 12 раундов):** 9 из 12 раундов завершаются `Winner=Draw` на `Dur=240s` — это лимит симуляции (`RoundSimLimit`), т.е. **бои не разрешаются за отведённое время**. Только 3 раунда (5, 8, 12) закончились решительно за 55–68с. Похоже на класс «голодание волн / одинокие мстители» (ADR-030) — юниты перестают добивать друг друга.
- Падение поймано через `UnityLogCheckDelegatingCommand` → вероятно, во время симуляции логируется незапланированная ошибка/исключение (Unity Test Framework валит тест при LogError), ЛИБО финальный assert по корреляциям не проходит из-за массовых ничьих.
- **Почему не починено в этой сессии:** на машине включён Smart App Control (до перезагрузки) — локальный Unity batch (`-runTests`) и любые свежесобранные .NET/нативные сборки блокируются (`0x800711C7`). Итерация только через CI (~10+ мин/прогон), чего для отладки боевой симуляции недостаточно эффективно. Лицензия UNITY_LICENSE в CI рабочая (активация успешна) — CI как таковой исправен.
