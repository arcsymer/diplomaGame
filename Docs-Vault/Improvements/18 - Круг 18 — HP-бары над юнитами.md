# Круг 18 — HP-бары над юнитами (читаемость боя в RTS)

Дата: 2026-06-23. Ветка `improve/circle-18` → main, тег `v1.18.0`.

## Improvement: world-space HP-бары над юнитами
Читаемость боя в RTS-виде — тезис «снижение микроменеджмента»: игрок считывает состояние боя с одного взгляда, не кликая каждый юнит. Прецедент — HP-бары над юнитами в StarCraft.
- **Что:** маленький горизонтальный HP-бар над головой юнита в RTS-камере (CurrentHp/MaxHp). Цвет green→yellow→red по доле HP (пороги 0.5 / 0.25).
- **Политика видимости (дёшево):** бар показывается только если юнит (a) выделен ИЛИ (b) повреждён (HP < max). Юниты на полном HP без выделения — без бара (меньше шума + дешевле). Все бары скрыты вне RTS-режима (TPS/герой) и для юнитов за экраном (viewport-куллинг).
- **Перф (ключевое — бои до 40+ юнитов):** ПУЛ из 48 виджетов, преаллок в Awake; никаких Instantiate/Destroy покадрово. Опрос видимости корутиной раз в 0.1 с; LateUpdate обходит только активные виджеты. Ноль аллокаций на горячем пути. Куллинг за экраном. Кэш Camera.main.
- **Код:** чистая статика `HealthBarLogic` (GetFillAmount / GetBarColor / ShouldShowBar / IsOnScreen / WorldToScreenBarPosition) — 32 EditMode-теста. `UnitHealthBarWidget` (виджет пула: fill + border Image). `UnitHealthBarSystem` (менеджер пула). Виджеты строятся ПРОГРАММНО (фон + Filled/Horizontal/Left fill + рамка) — префаб не нужен (ручное назначение в редакторе исключено, по конвенции).
- **Интеграция через Project Forge:** кнопка «Setup Unit Health Bars (C18)» + `ForgeBatch.SetupUnitHealthBars` (batch/CI), идемпотентно создаёт HealthBarsCanvas (Screen-Space Overlay, sortingOrder=20) + UnitHealthBarSystem на GameManagers, прошивает _modeController / _selectionSystem / _barCanvas.
