# Круг 4 — устранение GC-аллокаций боя и экран статистики матча

> Сессия 03 (2026-06-11). Фаза улучшения v3, круг 4. ADR-019, ADR-020.

## Состав круга

| # | Сфера | Улучшение | Приоритет |
|---|---|---|---|

## 1. GC-аллокации (ADR-019)

**Benchmark (SC2):** стабильный frame time без GC-спайков при сотнях юнитов.

**Найдено перф-ревью (стресс 20v20, before):** managedDelta до ~3.2 МБ/3300 кадров,
worst frame 15.6 мс. Главные источники: корутина VfxManager на каждую атаку
(до 40 аллокаций state machine за кадр), string-конкатенации HUD,
GetComponent в боевом кадре, GetActiveScene().name в аудио-тике,
GameObject.Find на спавне юнита.

**Реализация:** таймерные массивы вместо корутин VFX; кэш «цель — здание?» при захвате
цели; Unit.CachedCombat; TMP SetText-перегрузы + StringBuilder; кэш rally-позиций
и имени сцены. Публичные контракты не тронуты.

## 2. Экран статистики матча (ADR-020)

Таблица «Вы / Противник» на экране победы/поражения: убито врагов, потеряно,
урон нанесён/получен, кристаллов добыто, произведено юнитов, пик армии +
длительность матча. Архитектура: MatchStats (чистый класс, 11 EditMode-тестов) →
MatchStatsCollector (события Health.AnyDamaged, AnyDied, ResourceBank.CrystalsMined,
ProductionBuilding.UnitProduced) → MatchStatsView (uGUI). Герой и здания не считаются
в потерях юнитов. Forge: «Build Match Stats Panel» + ForgeBatch.SetupMatchStats.

## Верификация

| Метрика | Before (v1.3.0) | After (круг 4, v1.4.0) |
|---|---|---|
| EditMode-тесты | 217 | 228 (все зелёные) |
| PlayMode-тесты | 98 | 98 (все зелёные) |
| managedDelta, стресс 20v20 | 532–3208 КБ (~160–970 Б/кадр) | 594 КБ / 3353 кадра (~177 Б/кадр, стабильно) |
| Worst frame, мс | 15.64 | 3.8–4.8 |
| Avg frame, мс | 0.87–0.90 | 0.80–0.90 |
| Post-game статистика | нет | таблица 7 строк + длительность |

**Попутный фикс:** пост-ретритный скан ×5 (60 ед.) ловил стейлмейт «обе стороны целиком
отступили на базы» (диагональ между базами ~85) — расширен до ×10, завершаемость
гарантирована на всей карте.

**Открытое наблюдение (эскалация до P2):** накопленный винрейт зеркальных серий после
рандомизации фаз — 4:14 в пользу Enemy (p≈1.5% при честной монете). Остаточная
системная асимметрия существует, механизм не найден инспекцией кода — нужен
инструментированный Measure (длинная серия + лог первого убийства/ретритов) в круге 5+.
