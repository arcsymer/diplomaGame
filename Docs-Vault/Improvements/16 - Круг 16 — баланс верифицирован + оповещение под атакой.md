## Improvement: система оповещения «под атакой»
Тезис «снижение микроменеджмента»: игра сама направляет внимание игрока на угрозу, не требуя следить за всей картой.
- **Триггер:** `Health.AnyDamaged`, фильтр `Faction.Player` (Unit/Building/Hero). Урон врагу НЕ триггерит.
- **Реакция:** красный пульсирующий маркер на миникарте по мировой позиции (world→viewport через `MinimapController.MinimapCamera`), вспышка-виньетка по краю HUD, UI-звук (`AudioManager.PlayUiError`, CC0 Kenney — TODO заменить на профильный алерт-клип). Авто-сброс 3 с, троттлинг ~3 с (без спама в долгом бою).
- **Код:** чистая логика `UnderAttackAlertLogic` (ShouldTrigger/ShouldClear) без MonoBehaviour, 13 EditMode-тестов. MonoBehaviour `UnderAttackAlert` — подписка в OnEnable/OnDisable, без аллокаций на горячем пути, анимации корутинами (PrimeTween в проекте НЕТ — паттерн `UiPulse`).
- **Интеграция через Project Forge:** кнопка «Build Under Attack Alert (C16)» + `ForgeBatch.SetupUnderAttackAlert` (batch/CI), идемпотентно вшивает UnderAttackVignette→GameHUD, ThreatMarker→MinimapDisplay, UnderAttackAlert→GameManagers в `Sandbox.unity` через SerializedObject.
