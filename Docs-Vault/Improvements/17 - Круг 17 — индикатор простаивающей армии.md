# Круг 17 — индикатор простаивающей армии (idle army)

Дата: 2026-06-23. Ветка `improve/circle-17` → main, тег `v1.17.0`.

## Improvement: бейдж простаивающих юнитов
Вторая половина attention-routing UX (после оповещения «под атакой», круг 16). Тезис «снижение микроменеджмента»: игра показывает, какие юниты простаивают и ждут приказа, не заставляя игрока сканировать карту. Прецедент — индикатор idle worker в StarCraft.
- **Что:** HUD-бейдж в левом нижнем углу RTS_Block со счётчиком юнитов игрока в состоянии `Idle` дольше 5 с. Виден только при count > 0. Клик выделяет все эти юниты.
- **Логика порога:** только `UnitState.Idle`; `Holding` (приказ игрока) и `Patrolling` (самодостаточный цикл) ИСКЛЮЧЕНЫ. Часы простоя стартуют с первого наблюдения юнита (честно, без догадок о прошлом).
- **Цвет:** янтарный (0.85, 0.65, 0.05) — «требует внимания» (не красный = тревога, не зелёный = ресурс), по конвенции RTS.
- **Код:** чистая статика `IdleArmyLogic` (UpdateTimestamps / PruneDeadUnits / GetIdleUnits / CountIdleUnits) — 15 EditMode-тестов. MonoBehaviour `IdleArmyIndicator` — опрос `UnitRegistry.GetPlayerUnits` корутиной раз в 0.75 с (не покадрово), преаллоц. буферы (ноль аллокаций на опрос), клик → `SelectionSystem.SelectUnits`. Анимация — `UiPulse` (PrimeTween в проекте нет).
- **Selection:** добавлен public `SelectionSystem.SelectUnits(IReadOnlyList<Unit>)` (паттерн LoadControlGroup: ClearSelection → SetSelected → NotifyChanged).
- **Интеграция через Project Forge:** кнопка «Setup Idle Army Indicator (C17)» + `ForgeBatch.SetupIdleArmyIndicator` (batch/CI), идемпотентно вшивает IdleArmyBadge + CountLabel + IdleArmyIndicator в `Sandbox.unity`, бейдж скрыт по умолчанию.
