# Круг 20 — фидбек героя (хитмаркер прицела + готовность абилки)

Дата: 2026-06-23. Ветка `improve/circle-20` → main, тег `v1.20.0`.

## Сфера: HERO/TPS (впервые за сессию — до этого только RTS UX)
Measure/Identify по геройской/TPS-сфере (бенчмарк Diablo): фил-слой круга 12 силён (отдача, хит-флэш, нокбэк, хит-стоп, дэш-трейл), но **прицел не подтверждал попадания** — `HeroShooter.ShotFired(origin, end, bool hit)` поднимался, но UI флаг `hit` НЕ читал. Стрельба (до 6–7 выстр/с) была «немой». Самый острый разрыв. Тезис: прямое управление героем должно ощущаться весомо, чтобы оправдать уход из RTS-вида.

## Improvement (2 пункта)
- **A (главный) — хитмаркер прицела:** `CrosshairUI` подписан на `ShotFired`; при hit=true прицел вспыхивает тёплым оранжевым (1, 0.55, 0) + scale 1.0→1.15→1.0 за ~0.1 с; при промахе — только лёгкий scale-пульс (1.05), без цвета. Чистая логика `CrosshairHitmarkerLogic` (тестируема). `Time.unscaledDeltaTime` (играет в паузе/слоумо), interrupt-restartable (быстрая стрельба перезапускает с нуля). Тюнинг в `GameFeelSettings` (4 поля).
- **B (вторичный) — готовность абилки:** `AbilitySlotUI` ловит фронт cooldown >0→0 (`AbilityReadyLogic.DetectReadyEdge`), на нём `UiPulse.TriggerPulse()` + UI-звук (`AudioManager.PlayUiClick`). Игрок видит «абилка готова» не пялясь в HUD.
- 22 EditMode-теста чистой логики (Resolve / PingPong / IsCoolingDown / DetectReadyEdge + дефолты GameFeelSettings).
- Forge `SetupCrosshairHitmarker`: идемпотентно создаёт 4 Image-полоски прицела если нет, прошивает `_shooter`/`_settings`, **пишет дефолты в GameFeelSettings.asset** (через SetDirty + SaveAssets + ForceReserializeAssets — иначе новые SerializeField-поля на существующем ассете остаются type-default = чёрный; ревью поймало это до мержа), добавляет UiPulse на слоты абилок.
