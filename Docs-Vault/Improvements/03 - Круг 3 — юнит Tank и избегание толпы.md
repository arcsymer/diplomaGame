## 1. Tank (ADR-017)

**Benchmark (SC2):** сплеш-юниты (осадный танк) бьют по площади вокруг точки попадания;
контр-механика «дорогой AoE против дешёвой массы» — ядро камень-ножницы-бумага.

**Реализация:** полностью data-driven — `UnitData.AoeRadius` (0 = одиночная атака,
старый путь не тронут) и `UnitData.TargetPriority` (порядок кандидатов скана);
`CombatLogic.FindTargetsInRadius` (чистая статика, без аллокаций); производство —
новое здание WarFactory у обеих сторон (ИИ получает танки симметрично через
`EnemyCommander._enemyWarFactory`). Префабы/данные/сцена — кнопки Forge +
`ForgeBatch.SetupTank`.

**Баланс (старт):** HP 280 / Damage 25 / AoE 3.0 / Range 5 / CD 2.0 / Speed 3.0 /
Supply 3 / Cost 150 / ретрит отключён / приоритет целей — здания.

**Дефект, пойманный Balance-тестом до мёржа:** сплеш изначально центрировался на
танке — при Range 5 > AoE 3 выстрел с дистанции не накрывал неподвижное здание,
зеркальный бой танков замирал (`TankMirror_Resolves` падал по таймауту).
Исправлено: сплеш вокруг точки попадания (позиции цели), как в SC2.

## 2. Избегание толпы (ADR-018)

**Benchmark (SC2):** юниты не слипаются и не застревают; группа распределяется
вокруг точки приказа и вокруг цели атаки.

**Реализация (без нового кода агентов):**
- RVO-тюнинг NavMeshAgent: пехота radius 0.45 / HighQualityObstacleAvoidance /
  priority 50 / stoppingDistance 0.5; Tank radius 0.7 / priority 30 / stop 1.0.
- Формация: spacing 1.5 → 2.0; для групп >6 юнитов — 2.5.
- Отклонён boids-вариант: O(N²) и недокументированная запись `agent.velocity`.

**Анти-микро:** игрок по-прежнему отдаёт один приказ группе — распределение
происходит само (формационные смещения + RVO), APM не растёт.
