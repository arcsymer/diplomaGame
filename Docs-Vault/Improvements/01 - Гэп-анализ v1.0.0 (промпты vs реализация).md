## Найденные разрывы

### Р-1. Способности героя 2–4 — заглушки ⛔ (нарушение v2 DoD)
v2 §1/§9-M3: «стрельба, **2–4 способности**». Фактически реализован только Dash:
`AbilityType = { Dash, Placeholder2, Placeholder3, Placeholder4 }`,
`AbilitySystem.ApplyEffect` для слотов 2–4 пишет `Debug.Log` («placeholder»).
**Решение:** реализовать три реальные способности, усиливающие анти-микро-дизайн (ADR-013).

### Р-2. Baseline-метрики фазы улучшения не сняты ⛔ (v3 §8.2)
Нет ни перф-харнесса (FPS/frame time/GC), ни авто-плейтестов AI vs AI с винрейтами,
ни baseline-документа в `Improvements/`. Цикл Measure→…→Commit не может стартовать без точки отсчёта.
**Решение:** `PerfProbe`/`PerfStatsLogic` + Balance-тесты симметричных армий + baseline-документ (ADR-014).

### Р-3. Forge не расширен под фазу улучшения (v3 §5)
Нет кнопок: прогон харнессов, экспорт метрик улучшения, баланс-таблицы.
**Решение:** вкладка `Improve` + методы `ForgeBatch`.

### Р-4. Backlog без приоритетов (v3 §1.4)
`Improvements/00 - Backlog.md` — колонка «Приоритет» пуста, матрица impact×effort не применена.
**Решение:** проставить приоритеты, отсортировать.

### Р-5. Тултипы HUD — заявлены, но отсутствуют ⚠️ (v2 DoD «тултипы»)
В рантайм-UI нет tooltip-компонентов (есть только инспекторные `[Tooltip]`).
M6 в Dashboard помечен «тултипы ✅» — фактическая дыра в DoD.
**Решение:** занести в backlog с высоким приоритетом (честная фиксация), реализовать в следующем круге.
