# Круг 23 — направленный индикатор урона героя

Дата: 2026-06-23. Ветка `improve/circle-23` → main, тег `v1.23.0`.

## Improvement: направленность (завершает круг 21)
Круг 21 дал полноэкранную вспышку (fallback — `Health.AnyDamaged` не нёс источник). Круг 23 добавляет НАПРАВЛЕННОСТЬ: стрелка по краю экрана указывает на атакующего.
- **Backend (минимальный рипл, аддитивно):** новый `Health.AnyDamagedFrom` (`static Action<Health, float, Vector3>` — источник). Приватный `ApplyDamage` — единая точка: `AnyDamaged` поднимается ВСЕГДА (старые подписчики не тронуты: MatchStats / GameFeel / UnderAttackAlert / HeroDamageIndicator-flash), `AnyDamagedFrom` — только с источником. `TakeDamage(float, Vector3)` overload; `TakeDamage(float)` и `IDamageable` без изменений. `UnitCombat:502/574` передают `_unit.transform.position` (атака юнита, в т.ч. враг по герою).
- **Frontend (двухслойно):** `HeroDamageIndicator` подписан на оба — `AnyDamaged` (edge-flash, круг 21) + `AnyDamagedFrom` (стрелка, круг 23). Угол — чистая `HeroDamageIndicatorLogic.ComputeIndicatorAngleDegrees` (Vector3.SignedAngle по XZ; 0=спереди, +90=справа, 180=сзади). Референс — forward TPS-камеры (как в Halo/Division: «перёд» = куда смотрит камера). Стрелка вращается на угол, затухает ~1с, interrupt-restart. Только TPS.
- Forge `SetupHeroDamageIndicator`: создаёт HeroDamageArrow (центр+220px) + прошивает TPS Camera (подтверждено: не RTS) + damageArrowPeakAlpha.

## Верификация
- Компиляция чистая, Forge без исключений (TPS Camera прошита по имени). **EditMode 476/476** (464 + 12 углов-тестов: спереди→0 / сзади→180 / право→+90 / лево→−90 / диагонали). 0 регрессий — combat/Health-тесты зелёные после рефакторинга `ApplyDamage`.
- PlayMode (стрелка в сцене) — на CI.

## Бэклог / заметки
- Спрайт стрелки — дефолтный белый квадрат (вращается верно, но шеврон-спрайт читался бы лучше — нужен CC0-ассет/дизайнер).
- Forge берёт ПЕРВУЮ CinemachineCamera (сейчас одна = TPS, безопасно; при добавлении 2-й камеры — выбирать по имени).
- Остаток сферы: аудио (CC0); aim-spread (геймплей-риск power-fantasy); sprint-widen FOV (нужен sprint-state в HeroController API).
